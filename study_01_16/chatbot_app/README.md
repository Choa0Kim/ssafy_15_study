# 📰 AI 뉴스 분석가 프로젝트 (AI News Analyst Project)

사용자의 질문 의도를 파악하여 실시간 뉴스를 검색하고, 이를 논리적으로 분석하여 전달하는 지능형 AI 뉴스 파트너 서비스입니다.

---

## 🚀 1. 변경 및 추가된 주요 기능

### ⏹️ 답변 생성 중단 기능
* **사용자 편의성 강화:** 답변 생성 과정에서 실수를 발견하거나 다른 질문을 하고 싶을 때 즉시 중단할 수 있는 **중지(Stop) 버튼**을 추가하였습니다.

### 🔍 실시간 뉴스 검색 및 요약
* **의도 판별 알고리즘:** 사용자의 입력에서 뉴스 검색 의도를 자동으로 판별하여 최적화된 정보를 제공합니다.
* **실시간 데이터 수집:** Google News RSS URL을 통해 최신 기사 5개의 제목, 출처, 링크 정보를 XML 형태로 실시간 수집합니다.
* **구조적 분석:** LLM이 수집된 데이터를 바탕으로 단순 나열이 아닌 분석을 진행합니다.

---

## 🧠 2. 프롬프트 엔지니어링 (Prompt Engineering)

### 🎭 페르소나 및 역할 부여
* AI에게 **'지적인 AI 파트너 및 뉴스 분석가'**라는 명확한 페르소나를 부여하여 답변의 전문성과 톤앤매너를 일관되게 유지했습니다.

### 🏗️ 논리적 구조 강제
* 답변 시 **[배경 설명 - 상세 분석 - 결론]**의 3단계 논리 구조를 갖추도록 프롬프트를 설계하여 정보의 깊이를 대폭 개선했습니다.

### 🛡️ 제약 조건 및 가독성 최적화
* **검색 의도 분류:** 입력값이 뉴스 검색인지 일반 대화인지 판단하는 규칙을 설정했습니다.
* **출력 가이드:** "단순 링크 나열 금지", "마크다운 활용" 등의 제약 조건을 통해 사용자 가독성을 극대화했습니다.

## 🛠️ 3. 트러블 슈팅 (Troubleshooting)

### 1️⃣ 백엔드 연결 및 CORS 문제
* **문제:** 브라우저 보안 제약으로 인해 프론트엔드와 백엔드 간 통신이 불가능한 CORS 문제 발생.
* **해결:** Flask 서버 설정을 수정하여 외부 요청에 대한 접근 권한을 관리함으로써 연결 문제를 해결했습니다.

### 2️⃣ 모델 명칭 인식 및 비용 관리
* **문제:** `gpt-5-nano` 모델 인식 오류로 인한 서버 구동 불가 현상이 발생했습니다.
* **해결:** 서버가 공식적으로 지원하는 `gpt-4o` 모델로 변경하여 안정성을 확보했습니다.
* **비고:** `gpt-4o` 모델의 높은 크레딧 비용을 고려하여 추후 효율적인 토큰 관리를 계획 중입니다.

### 3️⃣ 기사 검색 결과 저장 기능 (진행 예정)
* **문제:** 뉴스 검색 결과를 Notion이나 DB에 저장하는 기능을 초기 개발 단계에서 숙지하지 못해 누락되었습니다.
* **향후 계획:** 다음 업데이트 시 외부 API를 연동하여 검색 결과를 영구 저장하는 기능을 추가할 예정입니다.

* **Language:** Python
