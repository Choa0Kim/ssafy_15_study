# 📂 Git & CLI 기초 정리 가이드

이 문서는 효율적인 소스코드 관리를 위한 필수 CLI 명령어와 Git의 작동 원리, 표준 워크플로우를 정리한 가이드입니다.

---

## 💻 1. 기본 CLI 명령어 (Command Line Interface)

터미널에서 디렉토리를 관리하고 파일을 제어할 때 사용하는 기본 명령어입니다.

* **`pwd`**: 현재 내가 작업 중인 폴더의 경로 표시 (Print Working Directory)
* **`ls`**: 현재 폴더의 파일 및 폴더 목록 확인
* **`ll`**: 목록을 상세 정보(권한, 날짜 등)와 함께 확인
* **`mkdir [폴더명]`**: 새로운 폴더(디렉토리) 생성
* **`touch [파일명]`**: 새로운 빈 파일 생성
* **`rm [파일명]`**: 파일 삭제
* **`rm -r [폴더명]`**: 폴더와 그 안의 내용물을 모두 삭제 (주의 필요)

---

## 🏗️ 2. Git 핵심 개념 (금고 비유)

Git은 소스코드를 관리하는 프로그램이며, **'금고'**라는 개념으로 이해하면 쉽습니다.

* **Repository (저장소)**: 소스코드가 저장되는 공간
* **Local Repository**: 내 컴퓨터에 있는 개인 금고
* **Remote Repository**: 서버(GitHub 등)에 있는 공용 금고 (24시간 가동)
* **Commit**: 현재 작업 내용을 금고에 저장하여 하나의 '버전'으로 만드는 동작
* **Push**: 내 컴퓨터 금고의 기록을 서버 금고로 업로드하는 동작
* **Pull**: 서버 금고의 최신 업데이트 내역을 내 컴퓨터 금고로 가져오는 동작
